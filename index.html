Executive summary

The brief in eight lines

  • ✦Six pages, but only a handful of operators and ONE repeatable playbook: novelty quiz ad -> harvest birth date/time/place -> paywalled 'personalized' reading -> recurring subscription.
  • ✦The core trick: everyone bills on a WEB Stripe funnel, NOT an App Store app. No 30% store cut, no store review, full control of price/trial/coupons. This is exactly the client's setup.
  • ✦Birth date + EXACT birth time + birth place are the three Kundli inputs. SkyWhisper, Aspectica, Lumius and Futurio all capture them inside a long psychographic quiz that doubles as data harvest and sunk-cost commitment.
  • ✦Going rate = a $0.99-$20 'trial' that silently rolls into $14.99/week (SkyWhisper), $39.99/mo (Aspectica) or $69.50/30-days (Lumius, ~$845/yr). Dark patterns: fake countdowns, 93%-off coupons, pay-what-you-want trial, spin-the-wheel discounts.
  • ✦Proven creative angles to copy: aura-color, past-life, soulmate-sketch, 'your chart knows why the same people keep changing your life', future-husband, cosmic-IQ, palmistry 'how many kids', witch-type.
  • ✦Recommended build: VedAstro (MIT-licensed, free/$1/mo, native MCP) computes the chart -> Claude Haiku writes the reading -> client's existing Stripe bills it. Chart cost ~$0, LLM cost a few cents per reading.
  • ✦Client's existing tools slot in cleanly: veo-flow-multi mass-produces the hook creatives per angle/language; vantage + Gonzo isolate the FB ad accounts against the near-certain bans.
  • ✦Real risk is operational, not technical: ad-account bans, chargeback/refund exposure from trial-auto-renew (HitMeal is 2.0 on Trustpilot despite 4.66 in-store), and thin offshore-shell reputation.

The build blueprint

The exact seven-stage pipeline to run this on autopilot with the client's existing stack.

The model in one line

A novelty FB/IG hook ad → a mobile quiz that harvests name + birth date + exact birth time + birth place → a Kundli API computes the real chart → Claude turns the chart JSON into a personalized reading → a Stripe cheap-trial paywall unlocks it → the trial silently rolls into a recurring subscription → email retention and cross-sell keep the LTV. Every operator in this teardown runs exactly this shape as a WEB funnel, never an app-store app — that is the whole trick: no 30% Apple/Google cut, no store review, and full control of the price/trial/coupon mechanics. The client already has Stripe, so he is one step ahead.

Stage 1 — Creative factory (autopilot)

Point veo-flow-multi at a prompt library of the winning angles (see the Winning Hooks table): natal-chart relationship-pattern, soulmate-sketch, future-husband, aura-color, past-life, palmistry "how many kids", cosmic-iq, witch-type, plus the calm-woman burnout/overthinking advertorial. Generate 20–40 image/video variants per angle. Lumius proves this scales to 15 languages off one funnel, so template each hook and swap the localized overlay text. The advertorials (lifestyle photo + long first-person story, no product) are the cheapest to produce and the least ad-like — run them alongside the flashy natal-wheel/orb creatives.

Stage 2 — Ad-account isolation

FB bans are a when, not if in this category. Use vantage to give each ad account its own browser fingerprint and each session a Gonzo residential proxy whose timezone/geo matches the target market. Warm accounts before spending; keep a rotating pool so one ban never stops the machine. Point every ad at a single quiz URL with UTM params (utm_source=Facebook, campaign/adset/ad macros) exactly as the studied funnels do.

Stage 3 — Quiz landing page (the data harvester)

Two build options:

  • Fast path: clone the FunnelFox pattern (Aspectica/Futurio) — a hosted SvelteKit quiz builder.
  • Full-control path (recommended): a bespoke Next.js app on Vercel, the Lumius pattern, with Supabase for funnel_events and PostHog for analytics.

The quiz is engineered as commitment + harvest. Order that all four capture funnels converge on:

  1. Gender + name.
  2. ~18–24 psychographic single-select questions with affirmation interstitials (this is the sunk-cost engine — it makes the eventual reading feel "earned").
  3. DOB wheel picker → birth-time picker (with an "I don't know" option) → birth place city autocomplete, geocoded to lat/long + timezone via Google Places/Mapbox. These are the three Kundli inputs.
  4. Animated "reading your chart / analyzing your energy" loaders with mid-loader upsell toggles.
  5. Email gate just before the wall (uid = base64(email) is the SkyWhisper trick — no password needed).
  6. A "your reading is ready" blurred teaser + fake countdown + injected discount, then the paywall.

Optional differentiator seen in Lumius: an on-device palm photo (MediaPipe hand-landmarker) for a palmistry add-on — more "personalization," but more friction/creepiness.

Stage 4 — Chart compute (Kundli API)

On email submit, POST {DOB, birthTime, lat, lng, tz} to the chart engine. Recommended: VedAstro — MIT-licensed (safe for a closed Stripe SaaS, unlike the AGPL engines), uses Swiss Ephemeris internally so the math equals the paid providers, free with the FreeAPIUser key or $1/mo for volume, and it ships a native MCP server so Claude can tool-call the chart, dashas and guna-milan directly. It returns the full Kundli as JSON: D1 + divisional charts, planetary positions, nakshatras, Vimshottari dashas, yogas, panchang, houses. If/when you need an SLA and support, graduate the compute layer to Prokerala ($19/mo, 100k credits) or FreeAstrologyAPI ($15/mo, 50k req) — keep everything else identical.

Stage 5 — AI reading (Claude)

Pass the chart JSON into Claude with a structured, persona-locked prompt: a fixed brand voice, a template of sections (core self / relationships / the recurring-pattern narrative the ad promised / the year ahead), and a hard rule to ground every sentence in the returned chart fields so the output is genuinely personalized and not AI-slop horoscope filler. Use Haiku for the per-reading volume path (a fraction of a cent each) and Sonnet for a premium tier. Render the result to a reading page and an optional PDF. Because the chart is real and the prose is chart-grounded, the client's honest differentiator vs the shells is that the "personalized" claim is actually true.

Stage 6 — Stripe subscription paywall

The client's existing Stripe is the entire monetization layer — Payment Element + Stripe Billing subscriptions + Google/Apple Pay express + a coupon. Copy the going mechanics:

  • A cheap intro: SkyWhisper's $0.99 → $14.99/week, Aspectica's $0.99 (3-day) → $39.99/mo, or Lumius's pay-what-you-want $6–$20.82 → $69.50/30 days (~$845/yr).
  • Pre-applied discount coupon, a 10–15 min countdown, a struck-through anchor, and (Futurio) a rigged spin-the-wheel that always lands on the same discount.
  • setup_future_usage=off_session so the card is stored for the auto-renew.
  • hCaptcha at checkout and Stripe Radar on to hold down the dispute rate.

Stage 7 — Retention / upsell (the real profit)

The trial charge barely covers ad cost; the money is in month 2+. Wire Stripe webhooks → an ESP (Klaviyo/Customer.io) to (a) deliver the reading and daily horoscope, (b) run dunning + a "3 days before your trial ends" reminder (this is also the compliance-safe version of the reminder Skool uses), (c) win-back cancelled users, and (d) cross-sell compatibility, ex-back, tarot, numerology and palmistry add-ons — Futurio's Tarot+Numerology recurring upsell and one-time guides are the template.

§09

Verdict

Is it worth building, the realistic economics, and the risks that actually decide it.

Is it worth building? — Yes, with eyes open.

The category is proven at scale. Aspectica runs ~4,000 library ads, Lumius ~2,200 across 15 languages, and Futurio/HitMeal roll up to AIBY (2B+ downloads). Multiple independent operators are spending continuously on the identical funnel — that only happens when the unit economics work. The client is uniquely well-positioned: he already has Stripe (the whole monetization layer), veo-flow-multi (the creative bottleneck others pay agencies for), and vantage + Gonzo (the ad-account survival kit). His honest edge over the shells: with a real Kundli API + Claude, the "personalized" claim is actually true, not generic AI filler.

Realistic economics

All figures below are reconstructed estimates from the observed funnels and each provider's public pricing — not the operators' actual conversion, retention, or margin data, which none of them disclose.

  • Revenue per subscriber: intro $0.99–$20, then $14.99/week (~$780/yr), $39.99/mo, or $69.50/30-days (~$845/yr) — if retained. Retention in this category is assumed weak; assume most churn within 1–2 cycles, so LTV is likely driven by the first 1–3 charges plus a long tail.
  • Cost to serve is trivial: chart ~$0 (VedAstro free/$1), reading a few cents (Haiku), Stripe 2.9%+30¢. On those inputs the marginal gross margin per reading is near-total (estimated).
  • The real cost is CAC. The entire game is whether blended trial-conversion × retention beats FB CPMs. That is a media-buying problem, not a build problem — the build is cheap and fast; the risk lives in the ad account and the churn curve.

Main risks

  1. Ad-account bans (highest-frequency risk). Astrology/"psychic" + clickbait quizzes get flagged; several funnels even run anti-bot tarpits and isolated infra. This is why vantage+Gonzo isolation is mandatory, not optional. Budget for constant account churn.
  2. Chargeback / refund exposure (highest-severity risk). The trial-auto-renew dark pattern is a dispute magnet: HitMeal is 2.0/5 on Trustpilot (86% 1-star) and 0/100 on ScamAdviser despite 4.66 in-store, with a consistent "charged after I cancelled / couldn't find the sub" complaint cluster. High dispute ratios get Stripe accounts frozen. Mitigate with a clear billing descriptor, a genuinely working one-click cancel, a pre-charge reminder email, hCaptcha + Radar, and a real refund SLA. Consider a backup MoR processor (Solidgate/Paddle) as the others do.
  3. App Store vs web checkout. This is the settled answer, not an open one: every funnel traced bills on WEB checkout (Stripe), not through an App Store — even AIBY (Futurio/HitMeal), which does publish App Store apps, runs its acquisition and billing on the web. Web avoids the 30% cut and store content-review of "psychic" claims. Keep an "entertainment purposes only" disclaimer (several of the traced funnels show exactly this wording) and stay web-first.
  4. Reputation / entity. The shells hide behind Estonia/Cyprus/UAE entities with thin footprints. The client should decide how close to that line he wants to sit; a cleaner entity + honest, chart-grounded readings + a working cancel flow is both lower-risk and a genuine differentiator.

Bottom line: technically a 1–2 week build (VedAstro + Claude + existing Stripe + existing creative/isolation tools) with near-zero marginal cost. The make-or-break is operational discipline on ad accounts and chargebacks, not the software.

§10

Open questions to settle before build

Eight decisions that shape the funnel — each trades conversion against chargeback and ad-account risk.

  1. Which geo(s) and language(s) launch first? Lumius proves 15-language scale off one app — where does the client already have payment coverage and ad-approval headroom?
  2. Vedic/Kundli or Western/tropical framing for the reading? This decides the API (VedAstro/Prokerala for Vedic vs Kerykeion for Western) and the creative angle set.
  3. How aggressive on dark patterns (fake countdowns, pay-what-you-want trial, 93%-off coupons, spin-wheel, cancel friction)? This directly trades conversion against chargeback and ad-account-ban risk.
  4. Trial economics: match SkyWhisper's $0.99 -> $14.99/week (bills more per year, churns faster) or Lumius's $6-$20 -> $69.50/mo? Pick the price ladder before building the paywall.
  5. What legal entity, billing descriptor, cancel flow and refund SLA? Every operator uses an EU/offshore shell — the client needs these three to survive Stripe Radar and disputes.
  6. Is the client's Stripe account prepared for this category's dispute rate, or does he want a backup MoR processor (Solidgate/Paddle) wired from day one like Futurio/HitMeal?
  7. What does the month-2+ retention product deliver (daily horoscope, transits, compatibility, tarot) to justify the recurring charge and cut refunds? The upsell ladder is where the profit is.
  8. Include the optional palmistry step (on-device palm photo, Lumius-style)? Extra 'personalization' and an upsell, but more friction and a creepier data ask.
